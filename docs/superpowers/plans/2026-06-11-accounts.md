# Accounts (Google + Apple Sign-In) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Optional sign-in with Google and Apple on web + iOS. Signed-in users get a backend identity (profile + editable display name) that Phases 3–5 build on. Guests keep today's exact experience; every sign-in surface is hidden unless its config is present.

**Architecture:** New `server/accounts/` TypeScript Lambda (esbuild bundle, mirroring `server/multiplayer/`) behind a new HTTP API Gateway v2, with a DynamoDB single table `UsersTable` and an HS256 session-JWT secret stored in SSM (SecureString). Clients obtain provider ID tokens natively (Google Identity Services on web; GoogleSignIn SPM + native `ASAuthorizationController` on iOS), POST them to `/v1/auth/login`, and hold a backend-issued 30-day JWT (web: `localStorage`; iOS: Keychain). Token verification adapts the proven `jose` code from `~/Personal/work/personal-brain/brain-server/src/auth/` (remote JWKS verify + HS256 session sign/verify).

**Tech Stack:** TypeScript + `jose` + `node:test` via `tsx` (server), vanilla TS (web), SwiftUI + GoogleSignIn SPM + AuthenticationServices (iOS), CloudFormation + bash deploy script (infra), `xcodegen` for project regeneration.

**Branch:** `accounts-auth` off `main`. Spec: `docs/superpowers/specs/2026-06-11-production-features-design.md` (Phase 2).

**Conventions:** imperative commit subjects, no `feat:` prefixes (match `git log`), HEREDOC commit messages, NO Claude co-author trailer. Minimize diff scope; match existing naming/patterns per package.

## Settled decisions encoded in this plan (do not relitigate)

- **JWT secret handling — documented choice:** the engine stack injects `API_KEY` as a plain env var via a NoEcho CFN parameter (`server/aws/template.yaml` lines 8, 68–69); CloudFormation does **not** support `{{resolve:ssm-secure}}` dynamic references for Lambda environment variables. So we stay consistent with the engine pattern: `deploy-accounts.sh` owns the secret — it generates and stores it in SSM (SecureString, `/chess-border/accounts/jwt-secret`) if missing, reads it back with `--with-decryption`, and passes it as a NoEcho CFN parameter that lands in the Lambda's `JWT_SECRET` env var. SSM is the durable store (redeploys never rotate the secret); the Lambda makes no SSM calls at runtime.
- **Audiences:** `GOOGLE_CLIENT_IDS` and `APPLE_CLIENT_IDS` are comma-separated env vars (web client ID + iOS client ID differ; Apple audience on iOS is the bundle ID `com.sahasraranjan.chessborder`).
- **User resolution:** `IDP#<provider>:<sub>` mapping wins; else a verified-email match links the new provider to the existing user; else create. `userId` = `crypto.randomUUID()` (Node 20 built-in, no dep).
- **Email lookup item:** the email-match step needs a lookup. We add an `EMAIL#<lowercased-email>` / `META` → `userId` mapping item (same shape as `IDP#`), written at user creation and at provider-link time. This avoids a GSI or scan and matches the existing single-table mapping pattern.
- **Apple caveats (must be encoded in code + tests):** Apple sends name/email **only on first authorization** — the handler must tolerate ID tokens with no `email` on later logins (lookup by `IDP#` sub succeeds, so it's a non-issue except when the mapping was lost; then we create a user with empty email and no email mapping rather than fail). Apple's `email_verified` may be the **string** `"true"`. Apple never puts the user's name in the ID token — clients pass an optional `name` hint in the login body (used only at creation). Private-relay emails are ordinary verified emails.
- **CORS:** handled by API Gateway HTTP API `CorsConfiguration` (allow `https://borderchess.org` + `http://localhost:5173`) — the Lambda never answers OPTIONS.
- **Web Apple sign-in is a stub:** hidden unless `VITE_APPLE_CLIENT_ID` is set (Apple Services-ID console setup is deferred). The server verifies Apple tokens from day one (iOS uses it).
- **Hidden-unless-configured:** web gates on `VITE_ACCOUNTS_API_URL` + `VITE_GOOGLE_CLIENT_ID` (same pattern as `VITE_MULTIPLAYER_WS_URL` in `web/src/ui/onlineGameView.ts:10`); iOS gates on Info.plist keys (`AccountsServerURL`, `GoogleClientID`) exactly like `MultiplayerConfig.isConfigured` in `ChessBorder/ChessBorder/Online/OnlineMultiplayer.swift:94-101`.

## API contract (single source of truth for all three packages)

```
POST /v1/auth/login   body {provider: "google"|"apple", idToken: string, name?: string}
                      → 200 {token: string, profile: Profile}
                      → 400 invalid body | 401 token verification failed
GET  /v1/me           Authorization: Bearer <token>
                      → 200 {profile: Profile} | 401
POST /v1/me           Bearer + body {displayName: string}   (1–30 chars after trim/collapse)
                      → 200 {profile: Profile} | 400 | 401
Profile = {userId, email, displayName, avatarUrl: string|null, createdAt: ISO string}
Errors  = {error: string}
```

DynamoDB items in `chess-border-users` (PK/SK):

```
USER#<userId>            / META   → {email, displayName, avatarUrl, createdAt, stats: {}}
IDP#<provider>:<sub>     / META   → {userId}
EMAIL#<lowercased email> / META   → {userId}
```

`stats: {}` is the Phase 3/4 placeholder — keep it an empty map now.

```ts
export type Provider = "google" | "apple";

export interface Profile {
  userId: string;
  email: string;
  displayName: string;
  avatarUrl: string | null;
  createdAt: string; // ISO 8601
}

export interface LoginRequest {
  provider: Provider;
  idToken: string;
  /** Display-name hint. Apple sends the user's name only client-side on
   *  FIRST authorization; clients forward it here. Used only at creation. */
  name?: string;
}

export interface LoginResponse {
  token: string;
  profile: Profile;
}

export interface MeResponse {
  profile: Profile;
}

export interface UpdateMeRequest {
  displayName: string;
}

export interface ErrorResponse {
  error: string;
}

const MAX_ID_TOKEN_LENGTH = 4096;

export function parseLoginRequest(raw: string | undefined | null): LoginRequest | null {
  const m = parseObject(raw);
  if (!m) return null;
  if (m.provider !== "google" && m.provider !== "apple") return null;
  if (typeof m.idToken !== "string" || m.idToken.length === 0 || m.idToken.length > MAX_ID_TOKEN_LENGTH) {
    return null;
  }
  const req: LoginRequest = { provider: m.provider, idToken: m.idToken };
  if (typeof m.name === "string" && m.name.trim().length > 0) {
    req.name = m.name;
  }
  return req;
}

export function parseUpdateMeRequest(raw: string | undefined | null): UpdateMeRequest | null {
  const m = parseObject(raw);
  if (!m || typeof m.displayName !== "string") return null;
  return { displayName: m.displayName };
}

/** Trim + collapse whitespace; valid at 1–30 chars, else null. */
export function validateDisplayName(name: string): string | null {
  const cleaned = name.replace(/\s+/g, " ").trim();
  if (cleaned.length < 1 || cleaned.length > 30) return null;
  return cleaned;
}

function parseObject(raw: string | undefined | null): Record<string, unknown> | null {
  if (!raw) return null;
  let data: unknown;
  try {
    data = JSON.parse(raw);
  } catch {
    return null;
  }
  if (!data || typeof data !== "object") return null;
  return data as Record<string, unknown>;
}
```

Cases (each `await`s `verifyIdToken(provider, token, ["client-a", "client-b"], keys)`):
1. Valid Google token (`iss: "https://accounts.google.com"`, `email_verified: true`, `email`, `name`, `picture`) → identity with lowercased email, name, avatarUrl.
2. Google `iss: "accounts.google.com"` (no scheme) also accepted.
3. Valid Apple token (`iss: "https://appleid.apple.com"`, `email_verified: "true"` **as a string**) → email present.
4. Apple token with **no email claim** → `email: null`, no throw (subsequent-login case).
5. `email_verified: false` → `email: null` (unverified email never used for linking).
6. Wrong audience → rejects. 7. Wrong issuer → rejects. 8. Expired token → rejects (sign with `setExpirationTime("-5m")`).
9. Audience matches the *second* configured client ID → accepted (multi-client-ID support).

- [ ] **Step 2: Implement `idtoken.ts`** — full file:

```ts
import { createRemoteJWKSet, jwtVerify } from "jose";
import type { JWTVerifyGetKey } from "jose";
import type { Provider } from "./protocol";

/** Identity extracted from a verified provider ID token. */
export interface IdpIdentity {
  provider: Provider;
  sub: string;
  /** Lowercased VERIFIED email, or null when absent/unverified.
   *  Apple omits email on logins after the first authorization. */
  email: string | null;
  name: string | null;
  avatarUrl: string | null;
}

/** Key sources per provider — injectable so tests use a local JWKS. */
export type ProviderKeys = Record<Provider, JWTVerifyGetKey>;

const ISSUERS: Record<Provider, string[]> = {
  google: ["https://accounts.google.com", "accounts.google.com"],
  apple: ["https://appleid.apple.com"],
};

/** Module-level remote JWKS: cached across warm Lambda invocations. */
export const REMOTE_KEYS: ProviderKeys = {
  google: createRemoteJWKSet(new URL("https://www.googleapis.com/oauth2/v3/certs")),
  apple: createRemoteJWKSet(new URL("https://appleid.apple.com/auth/keys")),
};

/** Verify a provider ID token's signature, issuer, audience, and expiry.
 *  Throws (jose errors) on any failure. */
export async function verifyIdToken(
  provider: Provider,
  idToken: string,
  audience: string[],
  keys: ProviderKeys = REMOTE_KEYS
): Promise<IdpIdentity> {
  const { payload } = await jwtVerify(idToken, keys[provider], {
    issuer: ISSUERS[provider],
    audience,
  });
  if (!payload.sub) throw new Error("id token missing sub");

  // Apple may serialize email_verified as the string "true".
  const emailVerified = payload.email_verified === true || payload.email_verified === "true";
  const email =
    emailVerified && typeof payload.email === "string" && payload.email.length > 0
      ? payload.email.toLowerCase()
      : null;

  return {
    provider,
    sub: String(payload.sub),
    email,
    name: typeof payload.name === "string" && payload.name.length > 0 ? payload.name : null,
    avatarUrl: typeof payload.picture === "string" ? payload.picture : null,
  };
}
```

```ts
import { SignJWT, jwtVerify } from "jose";

const ALG = "HS256";
const EXPIRES = "30d";

/** Issue a session JWT for a user. sub = userId; profile data stays in DynamoDB. */
export async function issueSession(secret: string, userId: string): Promise<string> {
  const key = new TextEncoder().encode(secret);
  return await new SignJWT({})
    .setProtectedHeader({ alg: ALG })
    .setSubject(userId)
    .setIssuedAt()
    .setExpirationTime(EXPIRES)
    .sign(key);
}

/** Verify a session JWT and return the userId. Throws on any failure. */
export async function verifySession(secret: string, token: string): Promise<string> {
  const key = new TextEncoder().encode(secret);
  const { payload } = await jwtVerify(token, key, { algorithms: [ALG] });
  if (!payload.sub) throw new Error("missing sub");
  return payload.sub;
}
```

```ts
import type { Provider } from "./protocol";

export interface UserRecord {
  userId: string;
  email: string; // "" when the provider never disclosed one (Apple edge case)
  displayName: string;
  avatarUrl: string | null;
  createdAt: string; // ISO 8601
  /** Phase 3/4 placeholder — empty map for now. */

}

export interface UserStore {
  getUser(userId: string): Promise<UserRecord | null>;
  putUser(user: UserRecord): Promise<void>;
  updateDisplayName(userId: string, displayName: string): Promise<void>;
  getUserIdByIdp(provider: Provider, sub: string): Promise<string | null>;
  putIdpMapping(provider: Provider, sub: string, userId: string): Promise<void>;
  getUserIdByEmail(email: string): Promise<string | null>;
  putEmailMapping(email: string, userId: string): Promise<void>;
}

/** In-memory store for tests. */
export class InMemoryUserStore implements UserStore {
  private users = new Map<string, UserRecord>();
  private idp = new Map<string, string>();
  private emails = new Map<string, string>();

  async getUser(userId: string): Promise<UserRecord | null> {
    const u = this.users.get(userId);
    return u ? structuredClone(u) : null;
  }
  async putUser(user: UserRecord): Promise<void> {
    this.users.set(user.userId, structuredClone(user));
  }
  async updateDisplayName(userId: string, displayName: string): Promise<void> {
    const u = this.users.get(userId);
    if (u) u.displayName = displayName;
  }
  async getUserIdByIdp(provider: Provider, sub: string): Promise<string | null> {
    return this.idp.get(`${provider}:${sub}`) ?? null;
  }
  async putIdpMapping(provider: Provider, sub: string, userId: string): Promise<void> {
    this.idp.set(`${provider}:${sub}`, userId);
  }
  async getUserIdByEmail(email: string): Promise<string | null> {
    return this.emails.get(email) ?? null;
  }
  async putEmailMapping(email: string, userId: string): Promise<void> {
    this.emails.set(email, userId);
  }
}

/** DynamoDB-backed store: one table keyed by (PK, SK), mirroring DynamoRoomStore. */
export class DynamoUserStore implements UserStore {
  private readonly doc: DynamoDBDocumentClient;
  constructor(private readonly tableName: string, client?: DynamoDBClient) {
    this.doc = DynamoDBDocumentClient.from(client ?? new DynamoDBClient({}));
  }

  async getUser(userId: string): Promise<UserRecord | null> {
    const res = await this.doc.send(
      new GetCommand({ TableName: this.tableName, Key: { PK: `USER#${userId}`, SK: "META" } })
    );
    if (!res.Item) return null;
    const it = res.Item;
    return {

          createdAt: user.createdAt,

    await this.putMapping(`EMAIL#${email}`, userId);
  }

      new GetCommand({ TableName: this.tableName, Key: { PK: pk, SK: "META" } })
```

```ts
import { randomUUID } from "node:crypto";
import type { IdpIdentity } from "./idtoken";
import type { UserRecord, UserStore } from "./store";

/** Resolve an authenticated provider identity to a user record:
 *  1. existing IDP mapping → that user;
 *  2. verified email matches an existing user → link this provider;
 *  3. otherwise create a new user.
 *  `nameHint` comes from the login body (Apple sends the name only
 *  client-side, on first authorization). */
export async function resolveUser(
  store: UserStore,
  identity: IdpIdentity,
  nameHint: string | undefined,
  now: Date
): Promise<UserRecord> {
  const byIdp = await store.getUserIdByIdp(identity.provider, identity.sub);
  if (byIdp) {
    const user = await store.getUser(byIdp);
    if (user) return user;
    // Mapping without a profile (partial delete): fall through and recreate.
  }

  if (identity.email) {
    const byEmail = await store.getUserIdByEmail(identity.email);
    if (byEmail) {
      const user = await store.getUser(byEmail);
      if (user) {
        await store.putIdpMapping(identity.provider, identity.sub, user.userId);
        return user;
      }
    }
  }

  const user: UserRecord = {
    userId: randomUUID(),
    email: identity.email ?? "",
    displayName: identity.name ?? nameHint ?? "Player",
    avatarUrl: identity.avatarUrl,
    createdAt: now.toISOString(),
    stats: {},
  };
  await store.putUser(user);
  await store.putIdpMapping(identity.provider, identity.sub, user.userId);
  if (identity.email) {
    await store.putEmailMapping(identity.email, user.userId);
  }
  return user;
}
```

```ts
let scriptPromise: Promise<void> | null = null;

function loadScript(): Promise<void> {
  scriptPromise ??= new Promise((resolve, reject) => {
    const s = document.createElement("script");
    s.src = GIS_SRC;
    s.async = true;
    s.onload = () => resolve();
    s.onerror = () => reject(new Error("Failed to load Google Sign-In"));
    document.head.appendChild(s);
  });
  return scriptPromise;
}

/** Render the official Google button into `parent`; resolves the callback
 *  with a Google ID token on each successful sign-in. */
export async function renderGoogleButton(
  parent: HTMLElement,
  clientId: string,
  onIdToken: (idToken: string) => void
): Promise<void> {
  await loadScript();
  const id = window.google?.accounts.id;
  if (!id) throw new Error("Google Sign-In unavailable");
  id.initialize({ client_id: clientId, callback: (resp) => onIdToken(resp.credential) });
  id.renderButton(parent, { theme: "filled_black", size: "large", shape: "pill" });
}

/** Best-effort: stop GIS from auto-selecting this account next visit. */
export function googleSignOut(): void {
  window.google?.accounts.id.disableAutoSelect();
}
```

```swift
import Foundation
import Security

/// Minimal generic-password Keychain wrapper for the session token.
enum KeychainStore {
    private static let service = "com.sahasraranjan.chessborder.auth"

    static func read(_ account: String) -> String? {
        let query: [String: Any] = [
            kSecClass as String: kSecClassGenericPassword,
            kSecAttrService as String: service,
            kSecAttrAccount as String: account,
            kSecReturnData as String: true,
            kSecMatchLimit as String: kSecMatchLimitOne,
        ]
        var item: CFTypeRef?
        guard SecItemCopyMatching(query as CFDictionary, &item) == errSecSuccess,
              let data = item as? Data else { return nil }
        return String(data: data, encoding: .utf8)
    }

    static func write(_ account: String, value: String) {
        let data = Data(value.utf8)
        let query: [String: Any] = [
            kSecClass as String: kSecClassGenericPassword,
            kSecAttrService as String: service,
            kSecAttrAccount as String: account,
        ]
        let update: [String: Any] = [kSecValueData as String: data]
        let status = SecItemUpdate(query as CFDictionary, update as CFDictionary)
        if status == errSecItemNotFound {
            var add = query
            add[kSecValueData as String] = data
            add[kSecAttrAccessible as String] = kSecAttrAccessibleAfterFirstUnlock
            SecItemAdd(add as CFDictionary, nil)
        }
    }

    static func delete(_ account: String) {
        let query: [String: Any] = [
            kSecClass as String: kSecClassGenericPassword,
            kSecAttrService as String: service,
            kSecAttrAccount as String: account,
        ]
        SecItemDelete(query as CFDictionary)
    }
}
```
